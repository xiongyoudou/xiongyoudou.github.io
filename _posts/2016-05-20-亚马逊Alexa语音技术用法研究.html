Alexa

Alexa包含两大模块：ASK与AVS。

ASK

ASK（Alexa Skills Kit）:ASK是一个包括自主服务API、工具、文档、代码示例的集合。可以通过build-in skills来实现比如播放音乐、回答常识问题、设定闹钟；也可以利用Smart Home Skill API来教Alexa怎样控制灯光和控制温度等；并且开发者可以为Alexa开发全新自定义的skills。

创建custom skills


根据需要选择Skill类型，此处选择Custom Interaction Model。


根据功能场景编写intents，然后根据intents编写相应utterance，尽量让utterance覆盖更多的语法结构，从而覆盖更多的使用场景。

Lambda

Lambda 是为custom skill最简单的创建云端服务的方式，是一个可以运行开发者上传代码的亚马逊web服务。开发者只需上传代码，Lambda管理其他所有的计算资源。上传至Lambda的代码可以用Node.js (JavaScript), Java, 和 Python编写，上传后可以在Lambda控制台进行测试。Lambda创建之后会生成Lambda ARN，将该ARN配置到custom skill的endpoint选项中。最重要的核心在于上传到Lambda的代码的编写，此处以js为例做说明如何在代码中控制返回的response：
开发者可以在代码中捕捉到intent识别后的回调：

TidePooler.prototype.intentHandlers = {
    "DialogTideIntent": function (intent, session, response) {
        var citySlot = intent.slots.City;
        var dateSlot = intent.slots.Date;
        if (citySlot && citySlot.value) {
            handleCityDialogRequest(intent, session, response);
        } else if (dateSlot && dateSlot.value) {
           ...
        } else {
           ...
        }
    },
    "AMAZON.StopIntent": function (intent, session, response) {
        var speechOutput = "Goodbye";
        response.tell(speechOutput);
    }
};

function handleWelcomeRequest(response) {
    var whichCityPrompt = "Which city would you like tide information for?",
        speechOutput = {
            speech: "<speak>Welcome to Tide Pooler. "
                + "<audio src='https://s3.amazonaws.com/ask-storage/tidePooler/OceanWaves.mp3'/>"
                + whichCityPrompt
                + "</speak>",
            type: AlexaSkill.speechOutputType.SSML
        },
        repromptOutput = {
            speech: "I can lead you through providing a city and "
                + "day of the week to get tide information, "
                + whichCityPrompt,
            type: AlexaSkill.speechOutputType.PLAIN_TEXT
        };

    response.ask(speechOutput, repromptOutput);
}

SSML

SSML(peech Synthesis Markup Language):当用户发起一个针对custom skill的请求时，你可以提供字符串让AVS转为speech，然而在某些情况下希望返回的response能够对text转语音有更多的控制，比如在speech中有一段很长的暂停、speech中加入很短的铃声等，因此有了SSML。SSML是提供某种方法来标记语音合成的标记语言，它支持一系列的标签。
audio
break
p
phoneme
s
say-as
speak
w
比如outspeech可以这样定义：

"outputSpeech": {
    "type": "SSML",
    "ssml": "<speak> Welcome to Car-Fu. 
    <audio src="https://carfu.com/audio/carfu-welcome.mp3" /> 
    You can order a ride, or request a fare estimate. 
    Which will it be?</speak>"
}

更详细SSML内容参照https://developer.amazon.com/public/solutions/alexa/alexa-skills-kit/docs/speech-synthesis-markup-language-ssml-reference

并且response中针对不同内容有相应的内容大小限制：

  • outputSpeech不能超过8000个字符
  • card中所有内容不能超过8000个字符
  • card中的image URL长度不能超过2000个字符
  • response总大小不能超过24kb

AVS

AVS（Alexa Voice Service）是指亚马逊的Alexa智能云端服务，它提供API让开发者集成语音控制功能到设备中，从而使得设备可以利用ASK来访问Alexa的内嵌功能以及第三方的skills。
API的Endpoint是：https://avs-alexa-na.amazon.com
可以支持的接口有：

Interface Summary
SpeechRecognizer AVS最核心接口，通过每个用户的语言表达识别事件
Alerts 设置、暂停、删除定时与闹钟
AudioPlayer 管理和控制语音的播放
PlaybackController 控制播放队列
Speaker 设备或者应用音量控制
SpeechSynthesizer Alexa发音
System 提供关于Alexa客户端信息

应用

  • 应用1：直接使用Alexa的build-in skills，如可以询问天气，询问音乐等，但是这一类场景最后只是简单的播放返回的音频流
  • 应用2：使用customSkill时必须要在每一次的语音中包含创建custom skill时填写的Invocation Name，然后AVS会自动接入该custom skill所运行的服务，并运行所上传的代码。最后返回音频流给设备。

疑惑：之前以为可以通过设置outspeech的type属性便可以控制客户端接收到的数据类型是text还是音频流，最后却又觉得不对劲。从字眼上看outspeech已经断定了返回的必须是speech了，至于type属性的设定则是控制返回的语音是通过纯文本(plaintext)还是SSML标记语言合成的差别而已。因此我并没有找到地方可以控制客户端能够接收到intent相关的任何文字信息。有找到其他开发者提出类似问题，亚马逊专业人员给出了解答：的确是不能直接返回text，但是有提供解决的思路：https://forums.developer.amazon.com/questions/10996/can-alexa-voice-service-return-text-in-the-respons.html
